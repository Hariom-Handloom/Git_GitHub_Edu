📘 Chapter 5: Making Changes, Staging, and Committing

🎯 Objective

In this chapter, you'll learn:

  • How to make changes to files
  • How Git detects those changes
  • How to stage and commit updated files
  • Common commands used in this process

🔁 What is a “Change” in Git?

A change refers to any modification made to a file tracked by Git:

  • Editing content
  • Adding new files
  • Deleting files
Git keeps track of what’s added, changed, or removed using a three-stage process:

Stage Description
Working Directory Where you modify files
Staging Area (Index) Where you prepare changes for a commit
Repository (HEAD) Where committed history is saved

📝 Step 1: Modify a Tracked File

Let’s say you previously created a file called hello.txt.
Open and edit it:

Hello, Git!
This is my first modification.

Now ask Git what changed:

git status

You’ll see:

Changes not staged for commit:
(use "git add ..." to update what will be committed)
modified: hello.txt

This means Git knows the file changed, but it hasn’t been staged yet.

➕ Step 2: Stage the Modified File

Staging tells Git you want to include the changes in the next commit.

git add hello.txt

Check status again:

git status

Output:

Changes to be committed:
modified: hello.txt

💾 Step 3: Commit the Changes

Once staged, commit them with a meaningful message:

git commit -m "Updated hello.txt with second line"

Now the repository has a new snapshot with the updated file.

📂 Step 4: Add and Commit New Files

Let’s say you create a new file called notes.md:

echo "Git is awesome!" > notes.md

Check status:

git status

You’ll see:

Untracked files:
notes.md

Stage and commit the new file:

git add notes.md
git commit -m "Added notes.md with initial content"

🗑️ Step 5: Deleting Files

If you delete a file manually (e.g., by deleting it in Explorer or using rm hello.txt), Git notices it:

git status

Output:

deleted: hello.txt

Stage and commit the deletion:

git add hello.txt
git commit -m "Deleted hello.txt"

Or do it in one step using Git:

git rm hello.txt
git commit -m "Deleted hello.txt using git rm"

🔍 Step 6: Viewing the Differences

You can preview what changed before staging:

git diff

To see what’s staged:

git diff --staged

To view commit history:

git log

🧠 Tip: Stage Multiple Files

You can stage all modified and new files at once:

git add .

Or selectively stage files:

git add file1.txt file2.md

⚠️ Common Mistakes to Avoid

Mistake Tip
Forgetting to stage changes Run git status often to check what's staged
Making poor commit messages Use short, clear messages like “Fix bug in login form”
Committing too much at once Break work into smaller logical commits

✅ Quick Summary

Command Purpose
git status See what changed
git add <file> Stage file for commit
git add . Stage all new/changed files
git commit -m "message" Commit staged changes
git diff Show file differences
git log View commit history

🔚 Conclusion

You're now comfortable with:

  • Editing files
  • Tracking changes
  • Staging and committing updates
These are the core skills of using Git effectively.
