‚ùì Definition

GitHub is a web-based platform that hosts Git repositories in the cloud. It provides:

  • A remote place to store your code
  • Tools for collaboration, issue tracking, and code review
  • Web interface to visualize repositories and commits
  • Integration with CI/CD and third-party tools

üìÅ Section 3: Creating a New Repository

What is a Repository?

  • A repository (or repo) stores your project files and their version history.
  • It acts as a project folder on GitHub.
Steps to create:
  1. On GitHub dashboard, click the "+" icon ‚Üí New repository
  2. Fill in:
    • Repository Name (e.g., my-first-project)
    • Description (optional)
    • Visibility: Public or Private
  3. Optionally, select:
    • Initialize with a README
    • Add a .gitignore template
    • Choose a license (for open-source)
  4. Click Create repository

üîó Section 5: Cloning a Repository

What is Cloning?

  • Cloning means copying a remote GitHub repository to your local system.
How to clone:
  1. Copy the repository URL (HTTPS or SSH) from GitHub
  2. Run in terminal:
git clone https://github.com/username/repo-name.git
  • This creates a local folder with the entire project and version history.

üöÄ Section 6: Pushing Local Repository to GitHub

When you create a project locally and want to upload it:

  1. Create a new empty repository on GitHub (no README)
  2. In your local project folder, link remote repo:

    3. git remote add origin https://github.com/username/repo-name.git
    git branch -M main
    git push -u origin main

    • This sets the remote named origin and pushes your local main branch.

üë• Section 7: Adding Collaborators

For private repositories:

  1. Go to your repository ‚Üí Settings ‚Üí Collaborators
  2. Enter GitHub username or email
  3. Send invite; collaborators must accept
  4. Collaborators can clone, push, and pull based on permissions

üß† Section 8: Summary

  • GitHub is a cloud hosting service for Git repositories with collaboration features
  • You must create an account to use GitHub
  • Repositories can be public or private, with flexible visibility
  • Cloning copies remote repos locally; pushing uploads local changes to GitHub
  • Collaborators enable teamwork on private repositories
