🧩 Section 1: Git Rebase – Streamlining Commit History

🔄 What is Git Rebase?

git rebase is used to move or combine a sequence of commits to a new base commit. It helps you create a cleaner, linear commit history.

🔍 Unlike merge, which creates a merge commit, rebase rewrites commit history.


📌 Basic Rebase Usage

git checkout feature-branch
git rebase main

This re-applies your feature branch commits on top of the latest main.


🔄 Rebase vs Merge

Merge Rebase
Keeps all commits Rewrites commit history
Adds a merge commit No merge commit (linear history)
Easier for tracking timeline Cleaner, linear history

⚠️ Caution

Do not rebase public branches (shared with others), as it changes commit history and may cause conflicts for collaborators.

💾 Section 2: Git Stash – Saving Work Temporarily

📦 What is Git Stash?

git stash temporarily saves uncommitted changes so you can switch branches without committing them.

💡 Helpful when you need to change context but aren’t ready to commit your work.


📌 Common Stash Commands

git stash       # Stash current changes
git stash list       # View stashed changes
git stash apply       # Re-apply the latest stash
git stash pop       # Apply and delete the stash


✅ When to Use Stash

  • You’re working on something but need to switch branches quickly
  • You’re not ready to commit, but don’t want to lose your changes

🧩 Section 3: Git Cherry-pick – Picking Specific Commits

🍒 What is Git Cherry-pick?

git cherry-pick allows you to apply specific commits from one branch to another.

git cherry-pick


✅ Use Cases

  • Apply a bug fix from one branch to another
  • Reuse a specific feature without merging the whole branch
  • Bring missed commits into a release branch

⚠️ Things to Keep in Mind

  • Creates a new commit with a new hash
  • Can lead to duplicate changes if overused carelessly

🔖 Section 4: Git Tags – Marking Important Commits

🔰 What is a Git Tag?

A tag marks a specific point in your Git history. It’s commonly used for releases.


📌 Creating and Listing Tags

git tag v1.0.0                     # Create a lightweight tag
git tag -a v1.0.0 -m "Release"     # Annotated tag
git tag                            # List all tags
git show v1.0.0                    # Show tag details

📤 Pushing Tags to Remote

git push origin v1.0.0
git push origin --tags       # Push all tags

✅ When to Use Tags

  • To mark stable releases (v1.0.0, v2.1.1, etc.)
  • To refer back to specific milestones
  • To enable users to download a specific version

💡 Section 5: Summary of Advanced Git Commands

Tool Purpose Example Use Case
rebase Rewrite history to a new base Clean up history before merging
stash Temporarily save work in progress Switching branches without committing
cherry-pick Apply selected commits from another branch Bring a fix into another active branch
tag Mark specific points in commit history Indicate official releases (v1.0.0)
