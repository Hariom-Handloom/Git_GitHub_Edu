📘 Chapter 4: Creating Your First Git Repository

📦 What is a Git Repository?

A Git repository (or "repo") is a special folder where Git tracks and manages versions of your files.

There are two types:

  • Local Repository: On your computer.
  • Remote Repository: Hosted on a platform like GitHub, GitLab, or Bitbucket.
In this chapter, we'll focus on creating and managing a local Git repository.

🔨 Step 1: Create a Project Folder

Start by creating a new folder for your project:

Example:
MyFirstGitProject/

You can do this using File Explorer or the terminal:
mkdir MyFirstGitProject
cd MyFirstGitProject

🧪 Step 2: Initialize the Repository

Inside the folder, initialize a Git repository using:

git init

✅ This command creates a hidden folder named .git, where Git stores all its tracking information (commits, branches, logs, etc.).
You’ll see:

Initialized empty Git repository in /your/path/MyFirstGitProject/.git/

Now your folder is officially under version control!

📄 Step 3: Create Some Files

Let’s add a sample file to the folder:

echo "Hello, Git!" > hello.txt

Now list the files:

ls

You should see:

hello.txt

But Git is not tracking this file yet.

🕵️ Step 4: Check Git Status

Run:

git status

You’ll see:

Untracked files:
(use "git add ..." to include in what will be committed)
hello.txt

This means Git sees the file but is not tracking it yet.

➕ Step 5: Add Files to Staging Area

To start tracking hello.txt, add it to Git’s staging area:

git add hello.txt

You can also add all files:

git add .

Now run:

git status

You’ll see:

Changes to be committed:
new file: hello.txt

This means the file is staged and ready to be saved (committed).

💾 Step 6: Commit Your Changes

To save a snapshot of your project, run:

git commit -m "Initial commit"

The -m flag lets you add a short message explaining what you’ve changed.
Now your first Git snapshot is saved!

📂 Step 7: View Your Commit History

Use this command to see your commits:

git log

You’ll see something like:

commit 3b65dfea...
Author: Your Name
Date: Fri May 16 13:00 2025

Initial commit

Press q to exit the log view.

🧹 Git File States Overview

File State Meaning
Untracked Git sees the file but doesn’t track it yet
Staged File is ready to be committed
Committed File is safely stored in Git history
Modified File has been changed since last commit

📘 Recap: Basic Git Workflow

  1. Create project folder
  2. Initialize Git → git init
  3. Add files → git add .
  4. Save snapshot → git commit -m "message"

🧠 Best Practices

  • Always write clear commit messages
  • Commit often: after every logical change
  • Use .gitignore to ignore unwanted files (explained in future chapters)

❓ FAQ

Q: What if I forget to use git init?
A: Git won’t track anything unless you initialize the folder.

Q: Can I delete .git folder?
A: You can, but it will erase all Git history and make the folder non-version-controlled again.

🔚 Conclusion

You’ve now created your first Git repository, added files, and made your first commit. This is the foundation of Git-based workflows.
