üîç What is Version Control?

Version Control is a system that records changes to a file or set of files over time so that you can recall specific versions later.

It allows you to:

  • Keep track of changes
  • Revert back to previous versions
  • See who made what changes and when
  • Work on the same files with others simultaneously
Think of version control as a "timeline" of your project. You can move forward, go back, or even split the timeline into alternate directions (branches).

‚úÖ 2. Revert Mistakes:

‚úÖ 4. Experiment Safely:

‚úÖ 5. Keep a History:

üß∞ Types of Version Control Systems

Version control systems come in different forms:

  1. Local Version Control
    • Only tracks changes on your computer.
    • No collaboration or sharing.
    • Example: RCS (Revision Control System)
  2. Centralized Version Control (CVCS)
    • There‚Äôs one central server.
    • All users connect to that server to get the latest changes or share their own.
    • Easy to manage but fails if the server crashes.
    • Examples: CVS, SVN (Subversion)
  3. Distributed Version Control (DVCS)
    • Every user has a full copy of the project, including the entire history.
    • No internet needed to make commits or view history.
    • More powerful and resilient.
    • Example: Git

Git is the most popular distributed version control system in the world.
