ğŸ“ Definition

A Pull Request (PR) is a GitHub feature that allows developers to propose changes from one branch (often from a fork) into another branch (typically the main or develop branch of the original repository). It enables:

  • Code review before merging
  • Discussion and feedback
  • Automated testing integration
  • A traceable history of contributions

ğŸ”‘ Pull Requests are central to collaboration in both open-source and team-based development.

ğŸ”§ Section 2: Creating a Pull Request

ğŸ“‹ Prerequisites

To create a PR, you should:

  • Have a fork or a local branch with committed changes
  • Push your changes to a remote branch on GitHub

ğŸ”¹ Step-by-Step: Creating a Pull Request

  1. Push your feature branch:

    git push origin your-branch-name

  2. On GitHub, go to your repository.
  3. Click "Compare & pull request" (if prompted)
    OR
    Go to the Pull Requests tab and click "New pull request"
  4. Choose:
    • Base branch: the branch you want to merge into (e.g., main)
    • Compare branch: the branch with your changes
  5. Add a clear title and detailed description.
  6. Click "Create pull request"

ğŸ” Section 5: Updating a Pull Request

If the reviewer requests changes:

  1. Make local edits
  2. Commit your changes:
    git commit -am "Refactor auth logic based on feedback"
  3. Push them to the same branch:
    git push origin your-branch-name
The PR will automatically update, and the reviewer will be notified.
